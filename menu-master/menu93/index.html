Firefox Padding Bug

To see the bug in action you need to be using Mozilla Firefox. For those of you without Firefox you can view a screenshot here. This issue occurs when an element/image has a height of 100% and no explicit width set inside of a block element that has padding on it with and a defined height. If you set the inner elements width to anything other than auto this issue goes away. Also, when changing box-sizing: border-box; the issue doesn't appear to apply except slightly in example 3.

For web developers, a temporary fix when you want your element/ image to be responsive to to it's container height is to make sure the elements parent container doesn't have any padding. Instead, apply the padding to the element itself as demonstrated in example 4.

These demos all use Bootstrap 3. For Bootstrap users see this related pen for several alternative demos on how to implement a better navbar image.
