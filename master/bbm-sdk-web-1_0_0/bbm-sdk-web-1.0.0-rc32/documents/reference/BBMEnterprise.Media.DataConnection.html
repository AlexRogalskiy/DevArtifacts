BBMEnterprise.Media.DataConnection

This class provides methods and events for managing a data connection.

A data connection is a real-time secure connection created as the result of a data-only call. Much like a voice/video call, the connection is established after the callee accepts the incoming request of the connection. See BBMEnterprise.Media.DataConnection#event:connected. And instead of transmitting voice/video data, any form/size of data is accepted.

A data connection can be initiated with BBMEnterprise.Media#createDataConnection and can be accepted upon the event BBMEnterprise.Media#event:incomingDataConnection for the incoming requests.

Data connection supports full duplex, i.e. both parties can send and receive freely at the same time.

Data connection is also reliable, i.e. the data transmitted whether over UDP or TCP will always be re-transmitted if lost. The ordering of the data is always maintained.

Each data connection can have multiple data channels. A data channel is created to group data sent with the same identifier/purpose. This identifier can be automatically generated by the data connection when omitted, or can be specified by the sender (which can be used to hint the receiver of the content sent).

To initiate a one-off data channel for a blob of data or a file, without having to worry about chunking, buffering, see BBMEnterprise.Media.DataConnection#send and see BBMEnterprise.Media.DataConnection#sendFile

To initiate a streaming like data channel, one can also create a data sender with BBMEnterprise.Media.DataConnection#createSender.

To receive, a data receiver is passed along to the client upon the event BBMEnterprise.Media.DataConnection#event:dataReceived, which is fired when the first data with a new data channel is seen by the data connection. See BBMEnterprise.Media.DataConnection.DataReceiver.

send(data, progress, id) → Promise

Sends the given string or Uint8Array over the data connection.

This function creates a data channel and manages sending the data over the channel in chunks to achieve maximum efficiency.

The receiver of the data will be typed based on the type of the data, i.e. BBMEnterprise.Media.DataConnection.DataReceiver.DataType.STRING if a plain string is supplied as data, or {@link BBMEnterprise.Media.DataConnection.DataReceiver.DataType.RAW if Uint8Array is supplied as data instead.

sendFile(file, progress, id) → Promise

Sends the given file over the data connection.

This function creates a data channel and manages sending the file over the channel in chunks to achieve maximum efficiency.

The receiver of the file will be typed as BBMEnterprise.Media.DataConnection.DataReceiver.DataType.FILE.
