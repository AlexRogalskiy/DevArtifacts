The possible registration failure reasons of a BBMEnterprise.Event.RegistrationChanged event.

Properties

Name Type Optional Description

GeneralFailure

 

 

The registration failed for unspecified reasons. All errors that are not directly actionable by the application will have this failure reason.

DeviceSwitchRequired

 

 

The registration failed because another client that supports only a single point of presence (SPoP) is currently registered with the infrastructure.

This account cannot be used with the BBMEnterprise Web SDK until such time that a persistent client that supports multiple points of presence (MPoP) has been registered and the current SPoP device has been deregistered.

RegistrationRevoked

 

 

Another client has superseded this client's registration with the infrastructure. This client is no longer registered and can no longer service requests.

This client must be re-setup before it can begin processing requests again. When this failure reason is issued, only a successful call to BBMEnterprise#setup can allow traffic to resume.

IMPORTANT: Do not call BBMEnterprise#setup automatically when this failure reason is reported as this can result in clients alternately being kicked off the infrastructure. It is recommended that the application notify the user of this condition and allow them to log-in again.
