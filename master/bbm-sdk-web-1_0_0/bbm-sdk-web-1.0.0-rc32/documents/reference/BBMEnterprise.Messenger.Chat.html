BBMEnterprise.Messenger.Chat

Provides a structure to encapsulate the details related to a chat. Each instance of a Chat is immutable.

Properties

chatId  string

The unique identifier of the chat. This is the chat identifier to use when identifying a chat to any of the Messenger functions. This value will never change.

invitePolicy  Chat.InvitePolicy

The policy that controls who may invite participants to the chat. This will only be defined when 'isOneToOne' is false because invites cannot be extended to others in a one-to-one chat.

isOneToOne  boolean

This will be set to true when the chat is a one-to-one chat between the local user and one other participant. This value will never change.

lastActivity  Date

The time of the most recent notable activity in the chat. Notable activity includes chat creation and new messages.

participants  Array of BBMEnterprise.Messenger.Participant

The list of participants that are currently in the chat. A chat may contain messages from past participants which are no longer in this list.

state  Chat.State

The current state of the chat.

A chat that is created in the current session by the local user will start in the Ready state.

A chat that is known to the SDK but has not yet been restored, will start in the Waiting state.

subject  string

The current subject of the chat. The local user may change the subject of a chat using the BBMEnterprise.Messenger#chatUpdate method.

When a subject change has been made by either the application or by another participant in the chat, a BBMEnterprise.Messenger#event:chatUpdated event will be issued.

State  string

The possible states of a BBMEnterprise.Messenger.Chat.

Properties

Name Type Optional Description

Waiting

 

 

The chat has been created and is waiting to be restored on demand. To restore the Chat, a call to Messenger.fetchChatMessages must be made by the application. Although, the chat has not been restored, new messages may still arrive via the Event.ChatMessageAdded event.

Restoring

 

 

The chat is being restored. A Event.MessageListUpdate event will be issued each time new data has become available.

Defunct

 

 

The infrastructure has indicated this chat is no longer available. This state exists so the user knows why their previously functional chat is no longer functional, and gives the user a chance to consume the content of the chat until they remove it. The SDK will not send or receive any further messages for a chat in this state. The SDK will not automatically remove defunct chats. The app may remove it explicitly via Messenger.chatLeave.

Ready

 

 

The chat has been fully restored. No further Event.MessageListUpdate events will be issued for this chat. Once a chat has reached this state, new messages will arrive via the Event.ChatMessageAdded event.
