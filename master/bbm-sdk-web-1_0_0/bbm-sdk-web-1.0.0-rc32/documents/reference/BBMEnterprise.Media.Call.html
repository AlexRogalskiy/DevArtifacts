Methods

accept() → Promise

Accepts the incoming call before ringing continues.

NOTE, Upon receiving the BBMEnterprise.Media#event:incomingCall, the client must accept this call before the call state advances.

This gives the client a chance to validate if the calling party is valid before presenting the call to the user.

If the call is found to be invalid, the client should reject BBMEnterprise.Media.Call#reject the call immediately.

Throws

BBMEnterprise.Media.Exceptions.InvalidStateError if called after the call is already accepted or answered, or if the call is not an incoming call.

Returns

Promise 

a Promise, which is resolved if successful, and may reject to the following errors:

  1. BBMEnterprise.Media.Exceptions.NotSupportedError, if the requested call options cannot be satisfied, e.g. video device is unavailable on the platform when the video option is enabled.

  2. BBMEnterprise.Media.Exceptions.PermissionDeniedError, if the media fails to be requested due to permission error. e.g. if the user denied the request to allow camera or mic to be accessed.

NOTE, When the caller has offered video, the answer can optionally also offer video. However, when the caller has not offered video, the answer must not offer video in the answer. One can then toggle video after the call is connected. This is a compatibility requirement for mobile clients.
