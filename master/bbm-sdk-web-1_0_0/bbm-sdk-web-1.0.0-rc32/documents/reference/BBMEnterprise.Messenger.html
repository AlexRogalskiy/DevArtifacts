SyncMode  string

The different modes supported for fetching chat content during the initial call to BBMEnterprise.Messenger#fetchChatMessages.

Properties

Name Type Optional Description

NoSync

 

 

Indicates that an initial sync of a chat should not sync anything more than the number of messages requested. In this mode, all messages up to the latest received message will be marked as delivered, even though there may be messages before what was requested that had not been delivered yet.

SyncToDelivered

 

 

Indicates that an initial sync of a chat should sync at least as far back as the earliest message which had not yet been delivered. In this mode, calling Messenger#chatMessageRead will mark all messages up to the specified message as read, even though there may be messages earlier than this that have not been delivered to any client.

SyncToRead

 

 

Indicates that an initial sync of a chat should sync at least as far back as the earliest message which had not yet been marked read by calling Messenger#chatMessageRead.

chatInvite(chatId, invitees) → Promise containing (Promise or Array of Promise)

Send an invitation to each user identified by their regId to participate in the chat identified the given chatId.

Parameters

Name Type Optional Description

chatId

string

 

The identifier for the chat for which invitations are to be issued. The must be a current participant in an existing chat to successfully invite others to join. The chat must not be a 1:1 chat, only conferences allow for the invitation of additional parties.

invitees

(string or Array of string)

 

This argument contains the regId(s) to which invitations to participate in the identified chat are to be sent. This argument will accept:

  • A regId when only a single user is to be invited.

  • An array of regIds when multiple users are being invited.

    Sending an invite to the regId of the local user is not supported and will result in the invite for this regId being rejected.
Throws

BBMEnterprise.Error.NotFoundError 

When the given chatId is not known to the SDK or the local participant is not a participant in the chat.

Returns

Promise containing (Promise or Array of Promise) 

When inviting a single user by passing a string as the invitees parameter, the promise returned will resolve to a single promise when the invitation has been sent.

When inviting multiple users by passing an array as the invitees parameter, the promise returned will resolve with an array of promises that will be resolved or rejected independently for each user invited to participate in the chat.

The promise may be rejected with any error. In this case, no invitations will have been sent.

fetchChatMessages(chatId[, options]) → Promise containing Array of BBMEnterprise.Messenger.ChatMessage

Retrieve the messages associated with a specified chat, including both messages that are cached locally, and possibly more retrieved from the server. This will initiate download of messages and resolve once the specified criteria are satisfied. If there are, a ChatMessageListUpdated event will be posted when more messages have been retrieved and are available to the client.

Examples 

// If the chat contains 1000 messages, and this function has not yet
// called for the chat, it will retrieve messages 950-1000. If it
// is called again with the same parameters, it will retrieve messages
// 900-950.
fetchChatMessages(chatId,{minFetchCount: 50,
                           minStatus: Messenger.SyncMode.NoSync})
// If the chat contains 1000 messages, and this function has not yet
// been called for the chat, and messages 0-800 had previously been
// delivered to another client on which the messages were read, then it
// will retrieve messages 800-1000. If it is called again with the same
// parameters, it will retrieve messages 750-800.
fetchChatMessages(chatId, {minFetchCount: 50})

Throws

Returns

Promise containing BBMEnterprise.Messenger.Chat 

A promise containing the chat metadata. It may be rejected if an error occurs in retrieving the metadata, or the chat does not exist, but it will be added to the filtered set even in these cases.

If an error occurs and the chat is only in the filter list, not the chat list, then it will not be possible to send messages in the chat. The two possible ways to enable sending of messages when in this state are to receive a message for the chat, or to add the chat to the filter list again, and receiving a resolved promise. The chat need not be removed before attempting to add it again.

filterRemove(chatId)

Remove chat from the filtered set synchronously. After this call, the chat metadata and messages can no longer be retrieved, and any incoming message for the chat will be dropped. Note, removing all chats from the filter list simply results in an empty filter list - it does not disable filter mode.

ChatMessageSendOptions  Object

An object to encapsulate the options to use when sending a message via the BBMEnterprise.Messenger#chatMessageSend function.

Properties

Name Type Optional Description

tag

ChatMessage.ReservedTag

 

Indicates the type of message to be sent.

content

string

Yes

Holds the plain text content of the message, when the 'tag' has such a concept; otherwise omitted.

If this field contains more than 2000 Unicode code points it will be truncated. The total size of the 'content' in UTF-8 is subject to a cumulative size limit when combined with the size of the 'data' object (encoded as JSON in UTF-8) and the size of the 'thumbData' attachment. These three fields combined must not exceed 71680 bytes (70 KB).

data

Object

Yes

This field contains opaque data managed by the app that is sent with the message. Unless specifically noted, the SDK does not examine or modify this JSON object. This can contain a single JSON object, or it can not exist at all. It cannot contain any other JSON type.

Many 'tag' values have a corresponding object defined as a child of the 'data' object with a key equal to the 'tag'. All 'tags' with such a corresponding child of 'data' require that child to be present. Other tags have no such requirement. The 'data' object can also contain other key-value pairs independently from the 'tag'.

The total size of the 'data' object (encoded as JSON in UTF-8) is subject to a cumulative size limit when combined with the size of the 'thumbData' attachment and the 'content' (also in UTF-8). These three fields combined must not exceed 71680 bytes (70 KB).

thumbData

(Uint8Array or Blob)

 

The raw bytes of a small file. The thumb data may contain no more than 56320 bytes (55 KB).

The total size of the thumb data is subject to a cumulative size limit when combined with the size of the 'data' object (encoded as JSON in UTF-8) and the size of the 'content' (in UTF-8). These three fields combined must not exceed 71680 bytes (70 KB).

fileData

Object

Yes

The data of a file to be sent out-of-band, with a reference to it included in the message. A file that is attached in this manner may be retrieved by calling download() on the message.

ChatStartOptions  Object

An object to encapsulate the options used to start a chat via BBMEnterprise.Messenger#chatStart.

Properties

Name Type Optional Description

subject

string

 

The subject of the chat. This may be empty. This field has a maximum size of 128 code points. Values larger than 128 code points will be truncated. If an existing 1:1 chat is chosen, and this subject differs, the existing chat's subject will be updated.

isOneToOne

boolean

Yes

When truthy, the chat is the singular 1:1 chat between the local user and the other party. The SDK ensures that only one such non-Defunct chat per remote party will exist at a time. Otherwise, the chat is a multi-party chat, even if there are fewer than two remote parties.

invitees

(Array of String or String)

Yes

The array of regIds of users being invited to join the chat once it has been created, or a single string to invite a single user. When isOneToOne is truthy, this list must contain a single regId, either in string form or as a single element array. Otherwise, it is optional or can be specified with 0 or more regId entries.

When a single string is passed, a single Promise will be returned for the invite, rather than an array of promises.

Creating a OneToOne chat with the regId of the local user is not supported. Creating a Conference where the regId of the local user is one of the invitees will result in the invite for this regId being rejected.

invitePolicy

Messenger.Chat.InvitePolicy

Yes

The policy that controls who may invite participants to the chat. This only applies when the 'isOneToOne' field is falsy. The default is BBMEnterprise.Messenger.Chat.InvitePolicy.ParticipantsOnly
